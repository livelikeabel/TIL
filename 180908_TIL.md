Functional Programming

- 선언적(declartive)이다.

- 알고리즘을 숨기려고 하고, 선언적으로 표현하면서 목표중심적으로 프로그램을 표현. 전역상태를 사용하지 않는다.



functional programming 

- higher-order function (일급함수)

- stateless (무상태)

- immutability (불변성)

- referential transparency (참조투명성, 순수성)



상태를 없애는것, 상태를 최소화 시키는 것이 제일 중요하다. 공유할 수 있는 범위를 작게 만드는 것.
